Aprendiendo a utilizar Git y GitHub

La verdad Git es una increible herramienta

Todo lo que e aprendido hasta ahorita sobre Git and GitHub

Que es Git?

Es un Sistema de control de versiones (VCS). Este tiene un historial de todo lo que se ha echo en el proyecto y quien lo ha echo...

Terminologia

Git:

Es un sistema de control de versiones.

REPOSITORIO:

Es un proyecto completo, una pagina web, la carpte del proyecto.

Commits:

Es cada uno de los cambios registrados en el historial de Git.

RAMAS:

Son ramificaciones nuevas, caminos nuevos que toma el proyecto .

En Git todo se trabaja por ramas, hay una rama principal que se le llama "MASTER" donde esta el proyecto que sale al publico, la version de la produccion.

Cada vez que se requiere trabajar en caracteristica nueva o corregir algo, se saca una rama, de tal manera que puedes trabajar en un ambiente AISLADO, es una COPIA EXACTA DEL PROYECTO pero separada.

CLON:

Es una copia exacta del repositorio cuando un programador se integra a un equipo de trabajo lo primero que se debe hacer se clona el repositorio en su equipo local.

FORK:

A diferencia de un clon y una rama, es un proyecto completamnete diferente que se crea a partir de otro, por ejemplo las distribuciones de linux. Un ejemplo claro son las distribuciones de linux, todas se basan en el mismo kernel, pero apartir de ahi cada una toma su diferente camino..... Por lo tanto cada que se vea que este proyecto es un fork de otro, es que se baso en el que habia trabajado este otro proyecto

HISTORIA

Fue creado por Linus Torvalds, Fue el creador tanto del kernel de linux, como de Git

HERRAMIENTAS

La herramienta mas importante para trabajar con git es la linea de comandos (TERMINAL)

NOTA: Todos los programas de interfaces sirven para agilizar el trabajo cuando ya se entiende como funciona GIT.

COMANDO MAS UTILIZADOS EN GIT

$ git init:

Se crea el repositorio inicial

$ git clone:

Se incia y clona a la vez un repositorio que ya existe

$ git status:

Ver el estado de tu repositorio

$ git add . O $ git add "nombre archivo":

Agrega todo el proyecto o solo el archivo selecionado al area de trabajo (Area de preparacion)

$ git commit -m: "Aqui va la descripcion del cambio realizado"

Este comando sirve para guardar los cambios en los que se quedo. La version a la que llego ese dia.

$ git rm --cached . O git rm --cached "Nombre del archivo:"

Revierte lo enviado al area de preparacion. Este codigo revierte lo que agregaste con get add .
